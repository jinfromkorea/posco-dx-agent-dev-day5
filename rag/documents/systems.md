# 사내 시스템 안내

## EP (Enterprise Portal)

### 설명
EP(Enterprise Portal)는 사내 표준 포털 시스템입니다. 모든 사내 시스템의 진입점 역할을 하며, 사내 표준환경의 개인 PC에는 바탕화면에 EP 바로가기 아이콘이 기본 제공되며, Internet Explorer(IE) 또는 Chrome 브라우저로 접속합니다. 사외에서 EP를 이용하려면 VDI를 통해 접속해야 합니다.

### 접속 방법
1. 바탕화면의 'EP' 바로가기 아이콘을 더블클릭 (사내 표준환경 PC 기본 제공)
   - 바로가기 아이콘은 IE 또는 Chrome으로 자동 실행됨
2. 또는 IE / Chrome 브라우저에서 https://swpsso.posco.net 직접 접속
   - EP ID(개인 ID)와 패스워드 입력
3. 이후 SSO 대상 시스템은 EP를 통해 자동 로그인
4. 사외 접속 시: VDI를 통해 접속 후 EP 이용 (VDI 항목 참고)
5. 문의: IT팀 내선 1000

## VDI (가상 데스크톱 환경)

### 설명
사외에서 사내 시스템(EP 등)에 접근하기 위한 가상 데스크톱 환경(VDI)입니다. EP ID와 패스워드로 로그인하며, 문자 인증(2차 인증)이 필요합니다. VDI 접속 후 사내와 동일하게 EP 및 연동 시스템(이러닝 등)을 이용할 수 있습니다.

### 접속 방법
1. 브라우저에서 https://pvdi.posco.net 접속 (PC 및 모바일 모두 가능)
2. EP ID(개인 ID)와 패스워드 입력
3. 문자 인증(SMS 2차 인증) 진행 - 등록된 휴대폰으로 전송된 인증번호 입력
4. 로그인 완료 후 VDI 화면에서 EP 및 각 시스템 링크/아이콘으로 접근
5. 문의: IT팀 내선 1000

## Microsoft Teams

### 설명
팀 채팅, 화상회의, 파일 공유를 지원하는 협업 도구. PC 및 모바일 모두 사용 가능하며, 웹 브라우저(Web 모드)와 데스크톱 앱(App 모드) 두 가지 방식으로 사용 가능. EP 메일 주소와 패스워드로 로그인합니다.

### 접속 방법

[웹 모드]
1. 브라우저에서 https://teams.microsoft.com 접속
2. EP 메일 주소 입력 후 EP 패스워드로 로그인

[앱 모드]
1. Microsoft Teams 데스크톱 앱 설치 (미설치 시 IT 헬프데스크 요청)
2. 앱 실행 후 EP 메일 주소 입력 후 EP 패스워드로 로그인
3. 앱 설치 요청: IT 헬프데스크(help@company.com)

## 이러닝

### 설명
사내 온라인 교육 수강, 이수 이력 관리, 자격증 등록 시스템. SSO 대상으로 EP 로그인 후 별도 인증 없이 이용 가능합니다. 사내에서는 EP의 앱을 통해서 접근할 수 있으나, 사외에서는 VDI를 통해서 접근해야 하며, VDI에서 제공하는 이러닝 링크로 접근하면 됩니다.

### 접속 방법

[사내 접속]
1. EP 포털(https://swpsso.posco.net) 로그인 후 EP 앱 목록에서 '이러닝' 아이콘 클릭
2. SSO 대상으로 별도 로그인 없이 자동 접속

[사외 접속]
1. VDI 로그인 후 VDI 화면에서 제공되는 이러닝 링크로 접속
2. SSO 대상으로 별도 로그인 없이 자동 접속

- 문의: 인재개발팀(edu@company.com)

## GitHub Copilot

### 설명
AI 코드 자동완성 도구로, IntelliJ 및 Visual Studio Code 등 개발 도구에서 플러그인으로 사용합니다. GitHub 계정으로 로그인하며, 계정은 EP ID에 '_poscodx'를 붙인 형식입니다. (예: EP ID가 'john'이면 GitHub 계정은 'john_poscodx')

### 접속 방법
1. IDE(IntelliJ 또는 Visual Studio Code)에서 GitHub Copilot 플러그인 설치
   - VS Code: 확장 마켓플레이스에서 'GitHub Copilot' 검색 후 설치
   - IntelliJ: Plugins 메뉴에서 'GitHub Copilot' 검색 후 설치
2. 플러그인에서 'Sign in to GitHub' 클릭
3. GitHub 계정으로 로그인: EP ID + '_poscodx' (예: EP ID 'john' → 'john_poscodx')
4. 브라우저에서 인증 완료 후 IDE로 돌아오면 자동 활성화
5. 문의: DX추진팀(dx@company.com)

## GitLab (그룹사통합망)

### 설명
그룹사통합망에 위치한 사내 소스코드 관리 및 CI/CD 파이프라인 운영 GitLab 인스턴스. SSO 미연동으로 별도 GitLab 계정이 필요하며, 아키텍처팀에서 EP ID로 계정을 발급합니다. GitLab 패스워드는 EP 패스워드와 무관합니다. 개발 직군 및 요청 시 타 직군도 접근 가능.

### 접속 방법
1. https://gitlab.company.com 접속
2. 아키텍처팀에서 발급받은 GitLab 계정(EP ID)과 GitLab 전용 패스워드로 로그인
3. GitLab 패스워드는 EP 패스워드와 무관 (EP 패스워드 변경과 무관)
4. 계정 발급 신청: 아키텍처팀에 이메일로 신청
5. 패스워드 분실 시: 아키텍처팀에 재발급 요청
6. 프로젝트 접근 권한 요청: 아키텍처팀

---

## Jenkins (그룹사통합망)

### 설명
그룹사통합망에 위치한 CI/CD 파이프라인 자동화 도구. 빌드, 테스트, 배포 작업을 자동화합니다. SSO 미연동, 공용 계정으로 접근합니다. 공용 계정은 아키텍처팀에서 발급합니다.

### 접속 방법
1. https://jenkins.company.com 접속
2. 공용 계정으로 로그인 (계정 정보: 아키텍처팀 문의)
3. 문의: 아키텍처팀
